TikTok Review Walkthrough

End-to-end mockup and live-web context for the TikTok integration.

This page is designed for app review. It explains how reviewers move from the public website to the separate live app console, then through TikTok authorization, session inspection, draft upload, and final creator review. The mockup video below is provided to make the flow clear even before using a sandbox account.

Included in this walkthrough

Public Website Live App Console Login Kit Session Check Draft Upload Publish Status
  • The Website URL is the public product home, not a login-only page.
  • The app console is separate and linked clearly from the public website.
  • The integration currently uses the safer draft upload path before final creator review in TikTok.
1

Start from the public website

Reviewers land on the public product site at the submitted Website URL. The site provides product overview, workflow explanation, legal pages, support links, and clear links into the review walkthrough and live app console.

2

Open the live app console and start Login Kit

The dedicated app console is on a separate page. From there, the reviewer or creator can click `Connect TikTok` to start authorization and return to the same console after the callback.

3

Inspect the connected session

After the callback, the app checks the current session and shows the connected account status and basic profile information in the live JSON panel.

4

Create the draft upload request

The app uses the public MP4 URL to initialize a Content Posting Upload API draft. The response includes the publish ID used for the status check.

5

Confirm status and finish review on TikTok

The current production scope stops at the safer draft stage. Final creator review and completion of the posting flow occur on the TikTok side.
